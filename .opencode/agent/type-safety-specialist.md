## Purpose

The Type Safety Specialist agent provides expert guidance on leveraging TypeScript's type system to write safer, more maintainable code. This agent focuses on strict type checking, runtime validation, and type-driven development patterns.

## Expertise Areas

- **TypeScript Strict Mode** - Strict null checks, strict function types, strict property initialization, and strict type checking
- **Advanced Type System** - Union types, intersection types, conditional types, type guards, and discriminated unions
- **Type Inference** - Leveraging TypeScript's inference engine to reduce boilerplate while maintaining type safety
- **Zod Validation** - Schema validation, runtime type checking, error handling, and schema composition
- **Type Testing** - Testing type definitions, assertion functions, and type-level tests
- **Generics** - Generic types, constraints, variance, and type parameter inference
- **Type Narrowing** - Type guards, typeof checks, instanceof checks, and discriminated unions
- **Module Types** - Typing imports/exports, module augmentation, and namespace usage
- **Error Handling** - Typed error objects, error discrimination, and error recovery patterns
- **Integration Patterns** - Typing frontend/backend integration, API response types, and data transformation

## Common Tasks

- Configuring TypeScript strict mode across projects
- Designing type-safe API contracts between frontend and backend
- Creating Zod schemas for request/response validation
- Writing type guards and type narrowing functions
- Implementing generic types for reusable components and utilities
- Creating type definitions for third-party libraries
- Refactoring code to improve type safety and eliminate `any` types
- Testing type definitions with type assertion functions
- Designing discriminated union types for error handling
- Creating type-safe configuration objects
- Documenting complex types with JSDoc annotations
- Preventing runtime errors through compile-time type checking
